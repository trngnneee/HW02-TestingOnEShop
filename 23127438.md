- **Input under test**: `name` in the personal profile update form/API `PUT /api/users/me`
- **Precondition**: the user is logged in with a valid JWT token
- **Postcondition for valid input**: the profile is updated successfully, and `GET /api/users/me` returns the new `name`

**Step 2 - Identify the domain and assumptions**

- A valid name must contain at least one real letter after trimming spaces.
- Vietnamese Unicode letters are valid.
- Spaces between name parts are valid.
- Digits, symbols, emoji, control characters, and code-like payloads are invalid for a human full name.

### **1.2 Phone Input**

#### Requirement 1: Domain Testing

**Step 1 - Identify the input variable**

- **Feature**: FR-04 Personal profile management
- **Input under test**: `phone` in the personal profile update form/API `PUT /api/users/me`
- **Precondition**: the user is logged in with a valid JWT token
- **Postcondition for valid input**: the profile is updated successfully, and `GET /api/users/me` returns the new `phone`
- **Postcondition for invalid input**: the system rejects the update, displays/returns a validation error, and the previous `phone` remains unchanged

**Step 2 - Identify the domain and assumptions**

FR-04 does not define a complete phone validation rule. Therefore, the following assumptions are used for designing domain test cases:

- Phone is required.
- A valid phone number must be a Vietnamese phone number.
- A valid phone number starts with `0` or `+84`.
- For local numbers starting with `0`, the value must contain only digits.
- For international numbers starting with `+84`, the `+` character is accepted only as part of the `+84` prefix, and the remaining characters must be digits.
- Letters, spaces inside the number, separators, unsupported symbols, missing values, and non-string values are invalid.

| P4 | Empty value | `` | Invalid | Phone is required |
